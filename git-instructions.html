Git Instructions for Accessing Laptop Ensemble Files

Cloning the Repository

You normally do this only once, to set up the lork folder on your computer. (Some day it may be necessary to delete your current lork folder and create a new updated one by cloning again.)

Using Terminal on the Mac

  1. Launch the Terminal application (in /Applications/Utilities).
  2. Type:
    git clone http://github.com/johgibso/lork.git
    followed by the return key.

    After doing this, your “lork” repository is in your home directory.

Using the GitHub Desktop app (Mac / Windows)

  1. Launch the GitHub Desktop application.
  2. On the Welcome page, which asks you to sign in, press “Skip this step” at the bottom.
  3. When you get to the main window of GitHub Desktop, which says “No Repositories Found,” press the Clone a Repository button on the right, click the URL tab in the window that appears, type in the URL field:
    http://github.com/johgibso/lork.git
    and press the Clone button. This will create a “lork” folder on your computer.
  4. Locate this folder using the Repository > Open in Explorer command (or Open in Finder, if you’re on a Mac).

Retrieving Updates

Launch Terminal (Mac), and type these two lines, each followed by the return key.

cd lork
git pull

Or in GitHub Desktop, use the Repository > Pull command.

Other Git Commands

Other commands, accessible in the Mac Terminal program, will be useful sometimes. These assume you are already in the lork directory (having given the “cd lork” command).

  • Learn the status of your lork repository...
    git status

    This will tell you about files that don’t match the repository copy stored at http://github.com/johgibso/lork. Note that this can show changes even if git pull says that you are “already up-to-date.”

  • Restore a file within your repository to the state on the web site.
    CAUTION: This will delete any changes you have made, for example to Auzzie preset files.
    cd subdirectory
    git checkout -- filename
  • Force entire local repository to be overwritten by remote repository.
    CAUTION: This will delete any changes you have made to your local files, though it should not delete files that are not part of the repository.
    git fetch --all
    git reset --hard origin/master

Shell Mini-Tutorial

Quick tutorial on navigating in the shell. It might help to compare this to the folder hierarchy you can see in the Mac Finder or Windows Explorer.

$ <— This is the command prompt. Other stuff will be to its right. 

$ cd lork               [go into the lork directory]
$ cd test/hemi-test     [go down two levels]
$ cd ..                 [go up one level]
$ cd                    [go to your home directory]
$ cd ~/lork             [go to a directory within your home directory]
$ pwd                   [find out what directory you’re in]
$ ls                    [list the files in the current directory]
$ ls -l                 [list in detail]
$ cd reper<tab>         [type tab key to auto-complete a name]

cd = Change Directory
pwd = Print Working Directory (“working” means “current”)
ls = List files in the current directory

Type the up-arrow key to access the history of commands you’ve given. When you find one you want, press the return key to execute it.

In Terminal (Mac), you can type “cd”, a space, and then drag a folder from the Finder into the Terminal window to print the full path to that folder. Then press return to go to that folder in the shell.
